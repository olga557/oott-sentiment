# OOTT Sentiment — дашборд сентимента нефтяного Твиттера

Ежедневный анализ твитов с хэштегом **#OOTT**: объёмы, вовлечённость, языки,
география, авторы, эмодзи, хэштеги, сентимент (влияние на цену нефти и
эмоциональный тон), темы дня и структурированное саммари (EN/RU).

## Структура

```
scripts/
  fetch_tweets.py   сбор твитов за день через twitterapi.io (ключ в .env)
  fetch_prices.py   дневные цены Brent (ICE front-month авто; Yahoo — fallback)
  ingest_excel.py   одноразовый бэкфилл из Excel
  make_batches.py   готовит батчи твитов дня для LLM-классификации
  merge_labels.py   валидирует и сливает метки в data/enriched/
  aggregate.py      считает все агрегаты для дашборда
  common.py         канонический формат твита
  locations.py      страна автора из поля location профиля
prompts/
  classify.md       контракт классификации (правила + схема меток)
  summarize.md      контракт саммари дня (схема JSON, EN+RU)
data/
  raw/              сырые твиты по дням (JSONL) — главный архив
  enriched/         твиты + метки {relevant, price_sentiment, emotional_sentiment, topics}
  summaries/        саммари дня (JSON по схеме summarize.md)
  batches/          рабочие файлы для классификации
  prices.json       цены Brent по дням
dashboard/          статичный дашборд (HTML+JS, ECharts), данные в dashboard/data/
run_daily.sh        ежедневный пайплайн (шаги без LLM)
publish.sh          публикация: коммит + пуш -> GitHub Pages
serve.sh            локальный просмотр дашборда
```

## Первый запуск

```bash
pip3 install -r requirements.txt
cp .env.example .env        # вставить ключ twitterapi.io
./serve.sh                  # дашборд на http://localhost:8765
```

## Ежедневный запуск

Механическая часть — скриптами, анализ — агентом Cursor:

1. `./run_daily.sh` — собирает вчерашние твиты, обновляет цены, готовит батчи,
   пересчитывает агрегаты. Все «механические» виджеты уже обновлены.
2. Агент (вручную в чате или через Cursor Automation):
   - классифицирует `data/batches/<день>_*.json` по `prompts/classify.md`
     (выход — JSONL-метки), **закреплённой моделью** — см. ниже;
   - `python3 scripts/merge_labels.py <день> <файлы меток>`;
   - пишет саммари по `prompts/summarize.md` в `data/summaries/<день>.json`;
   - `python3 scripts/aggregate.py <день>` — финальный пересчёт.
3. `./publish.sh` — публикует обновления на сайт
   https://olga557.github.io/oott-sentiment/ (коммит + пуш, дальше GitHub
   Actions разворачивает Pages автоматически).

Пример промпта для ежедневной автоматизации: «Запусти ./run_daily.sh в
~/Projects/oott-sentiment, затем классифицируй батчи дня по
prompts/classify.md (субагентом с закреплённой моделью), слей метки
merge_labels.py, сгенерируй саммари по prompts/summarize.md и пересчитай
aggregate.py».

## Закреплённая модель классификации

Метки должны быть сравнимы день ото дня, поэтому классификацию выполняет
всегда одна и та же модель (фиксируется в этом README и в промпте
автоматизации). Текущая закреплённая модель: **GPT-5.5** (субагент Cursor).
Модели Claude для классификации и саммари не используются (решение владельца).
Запасная модель при исчерпании лимита: Composer 2.5.

Калибровка от 2026-07-12 на дне 2026-07-11 (78 твитов):
GPT против Composer 2.5 — совпадение price_sentiment 84.6%,
emotional 87.2%; индекс цены +22.1 (GPT) против +30.8 (Composer).
GPT точнее на инверсиях («пожар потушен» = снятие угрозы), поэтому закреплён GPT.

При смене модели (напр., старую отключили): прогнать 2–3 старых дня новой
моделью, сравнить индексы со старыми значениями и зафиксировать расхождение
в этом файле.

## Смена LLM на внешний API (если понадобится)

Контракт классификации отделён от исполнителя: промпт в `prompts/classify.md`,
формат меток проверяет `merge_labels.py`. Достаточно написать скрипт, который
шлёт содержимое батчей + промпт в любой API и сохраняет тот же JSONL — весь
остальной пайплайн и дашборд не меняются.
